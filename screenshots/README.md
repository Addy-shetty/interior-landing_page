# Screenshots Guide

## How to Add Screenshots

To complete the README with visual examples, add the following screenshots to this folder:

### Required Screenshots:

1. **hero-desktop.png** - Desktop view of hero section
2. **services-desktop.png** - Services section on desktop
3. **portfolio-desktop.png** - Portfolio gallery with filters
4. **mobile-hero.png** - Mobile hero section
5. **mobile-menu.png** - Mobile navigation menu

## Tools to Capture Screenshots:

### Browser Built-in Tools:
- **Chrome DevTools:** 
  - Press F12 → Cmd/Ctrl+Shift+P → Type "screenshot" → Choose "Capture full size screenshot"
- **Firefox:** 
  - Right-click → "Take Screenshot" → Select area or full page

### Online Tools:
- **[Screenzy](https://screenzy.io/)** - Create beautiful mockups with device frames
- **[Screely](https://screely.com/)** - Add browser chrome and backgrounds
- **[Shots.so](https://shots.so/)** - Create stunning mockup images

### Desktop Apps:
- **[Responsively App](https://responsively.app/)** - Test and screenshot multiple devices
- **[Polypane](https://polypane.app/)** - Professional browser for responsive testing

### Browser Extensions:
- **Full Page Screen Capture** (Chrome)
- **Fireshot** (Chrome/Firefox)
- **Nimbus Screenshot** (Chrome/Firefox)

## Recommended Settings:

- **Resolution:** 1920x1080 for desktop views
- **Format:** PNG for best quality
- **Mobile:** Use iPhone 12/13 Pro size (390x844)
- **Compress:** Use [TinyPNG](https://tinypng.com/) to reduce file size

## Quick Capture Steps:

1. Visit your live site: https://majestic-bubblegum-cbd9ce.netlify.app/
2. Open DevTools (F12)
3. Toggle device toolbar (Cmd/Ctrl+Shift+M)
4. Select device size
5. Take screenshot
6. Save to this folder with appropriate name
7. Commit and push to GitHub
